1 • Real-Time Weather Signature Detection
SDG 11 — Target 11.5
Boyer–Moore · KMP
Arrays · Structures

Continuously scan long sensor code streams (wind/humidity/light) to detect hazardous patterns with low latency. Boyer–Moore provides fast skipping when patterns are rare; KMP guarantees linear worst-case performance. Arrays and small structures keep memory and indexing predictable for edge processors.

3 • IT Asset Lifecycle & E-Waste Routing
SDG 11 — Target 11.6
BST · Heap · MergeSort

Keep devices ordered by expiry to prioritise processing. A Binary Search Tree stores devices for range queries and ordered traversal; a Min-Heap gives the immediate next expiry for scheduling; Merge Sort is used for generating stable lifecycle reports.

5 • Market Price Analytics & Volatility
SDG 11 — Target 11.a
Fenwick · Segment Tree

Track frequent price updates per product. Fenwick Trees handle point updates and prefix sums efficiently; Segment Trees provide range min/max to detect spikes and support volatility windows. Combined, they power quick queries for dashboards and reports.

Precompute the full distance matrix using Floyd–Warshall on an adjacency matrix representation. This enables constant-time queries for cheapest routes between any two nodes in medium-sized logistics networks. BFS is used for simple connectivity checks and verifying unweighted segments.

10 • Faulty Batch Recall & Fraud Detection
SDG 11 — Target 11.5.3(b)
Rabin–Karp · Arrays · Lookup Table

Scan stored batch codes for suspicious signatures using Rabin–Karp rolling hash (fast window hashing). Arrays store codes compactly and lookup tables map batch id → warehouse metadata for immediate recall manifests. Brute-force verification is used for candidates to be exact.
