Business Use-Case Analysis — Arohanagara

Ten engineering-grade modules for a smart-city ecosystem. Each system links real business flows with data structures and algorithms implemented in Vinay/.

All code in Vinay/
1 • Smart Cloud Kitchen & Rider Dispatch
Hash Table · Heap
Dijkstra · Graph

Processes menu items and incoming orders using hash maps, keeps pending orders in a min-heap by promised time, and runs Dijkstra on a city road graph to compute shortest rider travel time from the kitchen to each customer.

Data Structures: unordered_map, priority_queue, adjacency list — Algorithms: Dijkstra
Orders → Hash & Heap → Dispatch Queue City Graph → Dijkstra → Rider Path

2 • Supermarket Dynamic Pricing Engine
Arrays · Hash Table
Heap · Sort

Stores products and price rules in arrays and hash maps, recomputes dynamic prices from demand and competitor inputs, and uses a max-heap to surface top-demand items while std::sort ranks listings by demand and adjusted price.

Data Structures: vector, unordered_map, priority_queue — Algorithms: std::sort
SKUs → Arrays / Hash → Dynamic Price Demand → Heap → Top-k Products

3 • Automated Billing Code Matching & Fraud Detection
Trie · Hash Table
KMP Pattern Match

Loads a master SKU registry into a Trie and hash map, reads each bill line by line, and validates that billed codes and descriptions match authorised products while KMP-based pattern checks flag suspicious or blacklisted phrases.

Data Structures: Trie, unordered_map, vector — Algorithms: KMP string matching
Bill Lines → Trie / Hash → Code Check Text → KMP → Fraud / Anomaly Flags

4 • Supplier–Buyer Allocation Platform (B2B Matching)
Hash Table
Heap · Sort · Greedy

Keeps supplier and buyer profiles in hash tables, uses std::sort and a custom Merge Sort to rank suppliers by rating and price, then builds a max-heap of compatible bids and greedily pops the highest-score pairs to create one-to-one supplier–buyer matches.

Data Structures: unordered_map, priority_queue, vector<Supplier> — Algorithms: std::sort, Merge Sort, greedy matching
Suppliers × Buyers → Score Matrix Scores → Max-Heap → Greedy Match Allocation

5 • Urban Event Ticketing & Crowd Flow Optimizer
Hash Table
Grid · BFS
Graph · Dijkstra

Uses hash sets and maps to validate ticket IDs and block reuse, represents the stadium interior as a 2D grid where BFS finds the safest path to the nearest gate, and models outside areas as a weighted graph where Dijkstra computes the least-cost route from each origin to the main gate.

Data Structures: unordered_set, unordered_map, 2D char array, adjacency list — Algorithms: BFS, Dijkstra
Ticket ID → Hash Tables → Valid / Duplicate Stadium Grid → BFS → Nearest Gate Path Road Graph → Dijkstra → Optimal Entry Route

6 • Government Document Authentication
Merkle Tree
Bloom Filter · Hashing

Issues digital documents by hashing their ID and content, inserts IDs into a Bloom filter for fast “may exist” checks, and periodically rebuilds a Merkle tree over all document hashes so that any presented document can be verified against the published root using a Merkle proof.

Data Structures: vector<Document>, BloomFilter, MerkleTree — Algorithms: string hashing, Bloom filter membership, Merkle proof verification
Doc(ID, Content) → Hash → Leaf Leaves → Merkle Build → Root ID → Bloom → Maybe → Verify Proof vs Root

7 • Government Complaint Routing
Trie · Hash Table
Heap · Dijkstra

Logs complaints with free-text descriptions, classifies them into types such as garbage or water using a Trie dictionary, runs Dijkstra on the ward graph to find the nearest capable response team, and for each team maintains a max-heap of assigned complaints ordered by severity and waiting time.

Data Structures: Trie, unordered_map, priority_queue, adjacency list — Algorithms: Dijkstra
Complaint Text → Trie → TypeTag Ward Graph → Dijkstra → Nearest Team Base Team → Priority Heap → Service Queue

8 • Smart Agriculture — Irrigation Network Planning
Kruskal MST
Union-Find

Registers farms with GPS coordinates, generates candidate pipe connections whose costs depend on distance and terrain, and applies Kruskal’s minimum-spanning-tree algorithm with a Union–Find structure to build a minimum-cost irrigation backbone that connects all farms.

Data Structures: vector<Farm>, vector<Edge>, UnionFind — Algorithms: Kruskal MST
Farm Coordinates → Dist & Terrain → Edge Costs Edges → Sort + DSU → Irrigation MST

9 • Citywide Job & Skill Matching Portal
Trie
Boyer–Moore
Heap

Indexes skill keywords in a Trie for prefix suggestions, stores candidate and job profiles, uses Boyer–Moore pattern matching to detect the required skill word in free-text fields, and for each job builds a max-heap that ranks candidates by match score minus distance between their areas.

Data Structures: Trie, unordered_map, priority_queue, vector — Algorithms: Boyer–Moore
Skill Dict → Trie → Prefix Suggestions Candidate Text → Boyer–Moore → Skill Match Job → Heap → Top-Ranked Candidates

10 • Dynamic Public Transport Timetable & Transfer Optimizer
Floyd–Warshall
Arrays · Binary Search

Represents hubs and direct routes as a weighted matrix, runs Floyd–Warshall to precompute all-pairs in-vehicle travel times, and for each directed edge stores a sorted list of departures so binary search can pick the next trip and assemble direct or single-transfer journeys with minimum arrival time.

Data Structures: adjacency matrix, vector<int> departures — Algorithms: Floyd–Warshall, binary search
Matrix → Floyd–Warshall → dist[i][j] departures[u][v] → lower_bound → Next Departure → Trip Plan (≤1 transfer)
