1 • Smart Cloud Kitchen & Rider Dispatch
Hash Table · Heap
Dijkstra · Graph

Simulates a cloud kitchen taking orders, scheduling preparation, and dispatching riders over a city graph. Hash maps store menu and orders, a min-heap prioritises promised times, and Dijkstra computes shortest rider routes between areas.

Data Structures: unordered_map, priority_queue, adjacency list — Algorithms: Dijkstra
Orders → Hash & Heap → Dispatch Queue
City Graph → Dijkstra → Rider Path

2 • Supermarket Dynamic Pricing Engine
Arrays · Hash Table
Heap · Sort

Manages a supermarket SKU catalog and recomputes dynamic prices based on demand and competitor inputs. Arrays hold products, hash maps cache current prices, a max-heap extracts top trending SKUs, and sort-based ranking orders listings.

Data Structures: vector, unordered_map, priority_queue — Algorithms: std::sort
SKUs → Arrays / Hash → Current Price
Demand → Heap → Top-k Products

Trie · Hash Table
KMP / Boyer–Moore

Reads digital bills line by line, validates SKUs against a master registry, and scans descriptions for suspect text. A trie and hash tables index valid codes, while pattern-matching functions flag forged or inconsistent lines before approval.

Data Structures: Trie, unordered_map, vector — Algorithms: string matching
Bill Lines → Trie/Hash → Code Check
Text → Pattern Match → Fraud Alert

Grid → BFS → Gate Path

6 • Government Document Authentication
Merkle Tree
Bloom Filter · Hashing

Issues documents with hashed fingerprints, inserts IDs into a Bloom filter, and periodically builds a Merkle tree. Verification recomputes the document hash and checks membership via Bloom and Merkle proof against the published root.

Data Structures: vector, Merkle levels, Bloom Filter — Algorithms: hashing
Doc → Hash → Leaf
Leaves → Merkle → Root
ID → Bloom → Exists? → Verify Proof

7 • Government Complaint Routing
Trie · Hash Table
Heap · Dijkstra

Logs complaints with descriptions, tags them by type using a Trie, and maps them to wards on a city graph. Dijkstra finds the nearest capable team base, and each team maintains a heap of assigned jobs ordered by severity and wait time.

Data Structures: Trie, unordered_map, priority_queue, adjacency list — Algorithms: Dijkstra
Text → Trie → Category
Ward Graph → Dijkstra → Nearest Team
Team → Heap → Work Queue

Union-Find

Models farms as nodes and potential pipe segments as weighted edges, then computes a minimum-cost backbone. Kruskal’s algorithm with union-find picks edges in ascending cost until all farms are connected in an MST.

Data Structures: vector<Edge>, Union-Find — Algorithms: Kruskal
Edges → Sort → Kruskal + DSU → Irrigation MST

10 • Dynamic Public Transport Timetable & Transfer Optimizer
Floyd–Warshall
Arrays · Binary Search

Represents hubs as a weighted matrix and runs Floyd–Warshall to compute all-pairs travel times. Each directed route stores a sorted departure list, and binary search picks the next available departure to build low-wait journeys.

Data Structures: adjacency matrix, vector — Algorithms: Floyd–Warshall, binary search
Matrix → Floyd–Warshall → Dist[i][j]
Departures[u][v] → lower_bound → Next Bus
